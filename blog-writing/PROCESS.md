# 블로그 포스팅 자동화 프로세스

---

## PHASE 1: 자료 수집 및 정리

### 입력값
- 스마트스토어 상품 URL (키워드 파일 불필요)

### 처리 순서
1. `python debug_smartstore.py <URL> <PRODUCT_ID>` → 상품 API 데이터 캡처
2. Claude Code가 상품 JSON 분석 → 상품명·카테고리·태그·옵션·상세설명 추출
3. 인터넷 트렌드 조사 → 네이버 자동완성 API + 웹 검색으로 연관 키워드 핫 이슈 발굴
4. **검색량 수치 사용 금지** — 실제 트렌드 근거(출처 명시)로만 키워드 우선순위 판단
5. 구매 페르소나 역설계 (페인 포인트 3가지 도출)

> ⚠️ 키워드 검색량은 Naver Keyword Planner 실측치가 있을 때만 기재.
> 없을 경우 "추정치" 표기 대신 트렌드 근거(언론·자동완성·SNS)로 대체.

### 산출물
```
output/{product_id}/
  product.json       # 정형 데이터
  product.md         # 블로그용 요약
  screenshot_*.png   # 상세이미지 원본
keyword_analysis.md  # 키워드 분석 + 페르소나
```

## PHASE 2: 블로그 글 작성

### 구조
```
5명의 페르소나 작가
    ↓ 각자 초안 작성
편집장 채점 (100점)
    ↓ 점수 + 피드백
페르소나 1차 수정
    ↓
편집장 최종 선택 (상위 1~2편)
    ↓
최종 본문 + 제목 3~5개 + 사진 배치 지시서 + 태그 10개+
    ↓
타겟층 시점 최종 검수
    ↓
검수 통과 시 → 확정 / 미통과 시 → 지적 항목만 수정 후 재확인
```

### 편집장 채점 기준 (각 25점)
| 항목 | 기준 |
|---|---|
| 분량 | 1,500~2,500자. 초과/미달 시 감점 |
| 키워드 | 핵심 키워드 3개 이상 자연스럽게 포함 |
| 사실성 | product.json 기반 검증 가능한 내용만 |
| 자연스러움 | 읽다가 "이거 광고네" 느낌 드는 문장 없어야 함 |

## PHASE 3: 타겟층 시점 최종 검수

### 목적
편집장 기준(형식·키워드·사실)을 통과한 글이 **실제 독자에게도 자연스럽게 읽히는지** 확인.
편집장이 못 잡는 맥락 오류(화자 시점 불일치, 흐름 단절, 구매 연결 누락)를 잡는 단계.

### 검수 방법
keyword_analysis.md에서 도출한 구매 페르소나 3명이 **실제 검색 후 유입된 독자 시점**으로 글을 처음부터 읽는다.

각 페르소나마다 아래 4가지를 확인:

| 체크 항목 | 질문 |
|---|---|
| 유입 적합성 | 내가 검색한 키워드로 이 글이 나왔을 때 클릭할 만한가? |
| 공감 지속 | 읽다가 "나 얘기 아닌데" 싶은 순간이 있는가? |
| 정보 충족 | 내가 궁금했던 것(어떤 야채, 얼마, 어디서)이 해결됐는가? |
| 행동 유도 | 다 읽고 나서 다음에 뭘 해야 할지 알 수 있는가? |

### 통과 기준
- 3명 중 2명 이상: 4개 항목 모두 OK → 확정
- 1개라도 NO가 나온 항목 → 해당 부분만 수정 후 재확인 (전체 재작성 없음)

### 자주 발견되는 오류 패턴
1. **화자 시점 불일치** — 글쓴이 경험이 독자 상황과 맞지 않음 (예: 첫째 이유식 독자에게 "둘째 때 알았다")
2. **흐름 단절** — 감성 서술 중 갑자기 표·스펙 나열로 전환
3. **구체성 부족** — 실제 어떤 제품을 샀는지, 아기 반응이 어땠는지 없음
4. **구매 연결 누락** — 감성 마무리 후 "그래서 어디서 사요?"에 대한 답 없음
5. **광고 패턴 문장** — "강추", "사용은 진짜 간단해요" + 바로 가격 나열
